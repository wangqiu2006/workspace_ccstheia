# 陀螺仪辅助循迹 集成说明

HLK-AS201 姿态传感器（陀螺仪）集成到八路灰度循迹小车，用于**直道抗抖 + 转弯平滑**。

> 核心原则：陀螺仪是**叠加增强层**。没接、掉线或关闭时，代码自动走纯灰度控制分支；pivot 仍由灰度回中和硬超时兜底，不会因陀螺仪缺失而卡死。

---

## 一、硬件接线

陀螺仪走 MCU UART0（`empty.syscfg` 中实例名 `UART_GYRO`）。

| HLK-AS201 | MSPM0G3507 | 说明 |
|-----------|-----------|------|
| UART_TX (引脚15) | **PA31** (UART0 RX) | 陀螺发 → MCU 收 |
| UART_RX (引脚16) | **PA28** (UART0 TX) | MCU 发 → 陀螺收 |
| VCC33 | 3.3V | 供电 |
| GND | GND | **必须共地** |

> ⚠️ TX/RX 是**交叉**接的（陀螺 TX 接 MCU RX）。接错则完全收不到数据。
> 通信参数 115200-8-N-1，由 SysConfig 配好，无需手动设置。

### 当前工程已占用引脚（勿冲突）
- AT8236 右轮：PWM1=PA0、PWM2=PA1（TIMA0，5V 容忍开漏输出）
- AT8236 左轮：PWM3=PA10、PWM4=PA11（TIMA1）
- 灰度：PB25 / PA25 / PA27（MUX选通）+ PB20（OUT）
- 编码器：左 A/B=PB23/PB27（U6/A电机/ENCA），右 A/B=PB11/PB14（U4/B电机/ENCB）；当前使用双边沿 x4 译码和 20ms 轮速 PI
- 蓝牙：PA21/PA22（MCU UART2；原理图网络名 TX3/RX3）
- 调试 SWD：PA19 / PA20

### 电气约束（上电前确认）

- PA0/PA1 为开漏输出。若 AT8236 模块自身没有上拉，PWM1/PWM2 必须各加 4.7k~10k 外部上拉，并根据 AT8236 的 VIH 选择 3.3V 或 5V 逻辑电源；不要依赖 MCU 内部弱上拉。
- PA10/PA11 按 3.3V 推挽 PWM 使用；AT8236 由 5V 供电时，应根据其数据手册确认 3.3V 高电平可被可靠识别。
- 编码器接口和灰度模块接口标有 5V 供电，而 PB23/PB27/PB11/PB14/PB20 应按非 5V 容忍输入处理。若模块为 5V 推挽输出，必须加电平转换或分压；若为开漏输出，只能上拉到 3.3V。

- PA10/PA11 也是天猛星板默认串口脚。用作电机 PWM 时，应确保板载 USB 串口已断开或处于高阻态，避免双输出争用。

## 三、陀螺仪如何帮助循迹

### 1. 直道抗抖（闭环驾驶段）
近直道时（`|err| ≤ GYRO_DAMP_ERRLIM`）用 Z 轴角速度 `gz` 做反向阻尼：`steer -= K·gz`。
车一旦有摆动趋势，就往反方向补差速把它按住，直线更稳。转弯时（err 大）自动不介入。
角速度先经过一阶低通和零偏死区；只有样本足够新且灰度线仍可见时才参与控制。

### 2. 转弯平滑（pivot 原地转段）
只在收到新的 20Hz 数据帧时，用相邻 yaw 的最短角差累计净转角，并处理 `-180°/180°` 跨界。
累计 yaw 达到 2° 后才确认转向符号；首个成功弯会学习“右转对应的 yaw 符号”，后续反向累计超过 6° 会安全退出且不计圈。
接近目标角前 `TURN_EASE_DEG` 度内线性减速；达到 `TURN_TARGET_DEG` 后还必须等灰度中间探头回中才提前结束。

> 陀螺掉线时使用“计时 + 中间回中”兜底；陀螺数据新鲜时必须达到目标角并回中。角度硬限、错误方向或超时只退出转弯，不提交圈数。

### 硬件上限（诚实说明）
启动时会按需确认订阅 `GYRO + ANGLE`、20Hz、主动上报，避免模块历史配置让融合静默失效。AS201 主动上报最高 **20Hz**（50ms/帧），所以：
- 直道阻尼是**低频修正**，对慢摆（1–3Hz）有效，压不住高频抖动。
- 转弯角度每 50ms 更新一次，减速输出会呈小台阶，但不会重复积分旧样本。

这是模块硬件限制，不是代码问题。

## 五、上车调试步骤

**第 1 步 — 架空验证编码器方向和标定**
蓝牙输出中的 `lc/rc=目标计数/实测计数/PWM`。给车轮前进命令时，左右实测计数都必须为正；否则调整对应 `ENCODER_*_SIGN`。再记录直道稳定时的实测计数，将 `ENCODER_COUNTS_AT_SPEED_MAX` 调到该值。`es` 位0/1表示左右反馈故障，位2/3表示左右闭环尚未建立；方向或信号异常时固件自动退回开环PWM，不会继续加力。

**第 2 步 — 验证降级路径**
先**不接**陀螺仪，确认车辆仍能循迹和完成转弯（陀螺掉线 → `gyro_ok=0` → 控制走纯灰度分支）。
也可把 `GYRO_ENABLE` 设 `0` 编译，效果相同。

**第 3 步 — 接上陀螺仪**
按第一节接线，重新跑。观察：直道是否更稳、转弯是否更平滑。

**第 4 步 — 按现象调参**
- 直道**反而更抖** → 把 `GYRO_DAMP_SIGN` 改成 `-1`（符号取决于陀螺装的正反，这是最可能要改的一处）。
- 直道稳了但**反应发钝、过弯迟疑** → 调小 `GYRO_DAMP_K`。
- 转弯**转过头** → 调小 `TURN_TARGET_DEG`；**转不够或低速停顿** → 先调大 `PIVOT_STEER_MIN`，再按需要调大 `PIVOT_STEER`。

## 六、故障排查

| 现象 | 排查方向 |
|------|---------|
| 接了陀螺但无效果（一直走纯灰度） | TX/RX 是否交叉、是否共地、`UART_GYRO` 在 SysConfig 里是否无红叉 |
| 完全收不到数据 | 波特率是否 115200、陀螺是否上电（模块 LED） |
| 数据帧收到但被丢弃 | 看 `len_mismatch_count` 和 `frame_error_count`；前者表示订阅/长度不匹配，后者表示非法类型或帧格式 |
| 接收缓冲溢出 | 看 `g_gyro.overflow_count`；UART ISR每次最多搬运8字节、主循环每次最多解析32字节，正常20Hz上报不应溢出 |
| 编译找不到 `UART_GYRO_INST` | 打开 `empty.syscfg` 存盘一次，让 SysConfig 重新生成 `ti_msp_dl_config.h` |

> 诊断计数器（`frame_count` / `len_mismatch_count` / `frame_error_count` / `overflow_count` / `checksum_err_count`）
> 都在 `g_gyro` 句柄里，调试时用 CCS 的 Expressions 窗口直接看即可。

## 七、集成改了哪些地方（备查）

- `empty.syscfg`：保持为引脚/外设配置源；蓝牙由生成的 `UART_BT_INST` 自动解析到当前 UART2 / PA21-TX / PA22-RX，不在应用代码硬编码串口号。
- `empty.c`：
  - 加 `#include "hlk_as201.h"`、陀螺参数区、运行时状态全局量。
  - 新增 `UART_GYRO_INST_IRQHandler`（收字节进环形缓冲）与 `gyro_update()`（消费完整帧 + 在线判定）。
  - 新增编码器 `GROUP1_IRQHandler`、x4译码、20ms双轮PI与故障回退；pivot state 2 旁路PI。
  - 主循环调用有预算的陀螺解析；闭环驾驶段消费缓存阻尼；pivot state 2 加角度积分平滑收尾。
  - `main()` 里 `AS201_Init` + `AS201_EnsureReportConfig`（按需配置并回读验证，未接陀螺则超时后照常运行）。
- 保留 `df1cb80` 的圈数计数、pivot 状态机、灰度扫描和 250Hz 控制时序；针对当前板卡适配双 PWM 定时器、跨 GPIO 端口 MUX 和 AT8236 极性。
