# 陀螺仪辅助循迹 集成说明

HLK-AS201 姿态传感器（陀螺仪）集成到八路灰度循迹小车，用于**直道抗抖 + 转弯平滑**。

> 核心原则：陀螺仪是**叠加增强层**。没接、掉线、或关闭时，代码自动退回原纯灰度循迹，行为与集成前完全一致。不改动任何原有循迹逻辑。

---

## 一、硬件接线

陀螺仪走 MCU UART0（`empty.syscfg` 中实例名 `UART_GYRO`）。

| HLK-AS201 | MSPM0G3507 | 说明 |
|-----------|-----------|------|
| UART_TX (引脚15) | **PA31** (UART0 RX) | 陀螺发 → MCU 收 |
| UART_RX (引脚16) | **PA28** (UART0 TX) | MCU 发 → 陀螺收 |
| VCC33 | 3.3V | 供电 |
| GND | GND | **必须共地** |

> ⚠️ TX/RX 是**交叉**接的（陀螺 TX 接 MCU RX）。接错则完全收不到数据。
> 通信参数 115200-8-N-1，由 SysConfig 配好，无需手动设置。

### 当前工程已占用引脚（勿冲突）
- AT8236 右轮：PWM1=PA7、PWM2=PA12（TIMA0，推挽输出）
- AT8236 左轮：PWM3=PA10、PWM4=PA11（TIMA1）
- 灰度：PB25 / PA25 / PA27（MUX选通）+ PB20（OUT）
- 编码器：左 A/B=PB23/PB27，右 A/B=PA18/PB14
- 蓝牙：PA21/PA22（MCU UART2；原理图网络名 TX3/RX3）
- 调试 SWD：PA19 / PA20

### 电气约束（上电前确认）

- PA7/PA12/PA10/PA11 均按 3.3V 推挽 PWM 使用；AT8236 由 5V 供电时，应根据其数据手册确认 3.3V 高电平可被可靠识别。
- 编码器接口和灰度模块接口标有 5V 供电，而 PB23/PB27/PA18/PB14/PB20 应按非 5V 容忍输入处理。若模块为 5V 推挽输出，必须加电平转换或分压；若为开漏输出，只能上拉到 3.3V。
- PA10/PA11 也是天猛星板默认串口脚。用作电机 PWM 时，应确保板载 USB 串口已断开或处于高阻态，避免双输出争用。

## 三、陀螺仪如何帮助循迹

### 1. 直道抗抖（闭环驾驶段）
近直道时（`|err| ≤ GYRO_DAMP_ERRLIM`）用 Z 轴角速度 `gz` 做反向阻尼：`steer -= K·gz`。
车一旦有摆动趋势，就往反方向补差速把它按住，直线更稳。转弯时（err 大）自动不介入。
角速度先经过一阶低通和零偏死区；只有样本足够新且灰度线仍可见时才参与控制。

### 2. 转弯平滑（pivot 原地转段）
只在收到新的 20Hz 数据帧时，用相邻 yaw 的最短角差累计净转角，并处理 `-180°/180°` 跨界。
首个有效 yaw 增量自动确定本次转向符号，反向晃动会抵消而不会被重复累计。
接近目标角前 `TURN_EASE_DEG` 度内线性减速；达到 `TURN_TARGET_DEG` 后还必须等灰度中间探头回中才提前结束。

> 原有"计时 + 中间回中"和硬超时仍作为兜底；陀螺掉线不会让转弯卡死。

### 硬件上限（诚实说明）
启动时会按需确认订阅 `GYRO + ANGLE`、20Hz、主动上报，避免模块历史配置让融合静默失效。AS201 主动上报最高 **20Hz**（50ms/帧），所以：
- 直道阻尼是**低频修正**，对慢摆（1–3Hz）有效，压不住高频抖动。
- 转弯角度每 50ms 更新一次，减速输出会呈小台阶，但不会重复积分旧样本。

这是模块硬件限制，不是代码问题。

## 四、可调参数（都在 `empty.c` 参数区，标 ★ 的最常调）

| 参数 | 默认 | 说明 |
|------|------|------|
| `GYRO_ENABLE` | `1` | 设 `0` 编译期完全禁用陀螺仪，一行回到原版 |
| ★ `GYRO_DAMP_K` | `6` | 直道阻尼增益。越大越稳但反应钝，`0` = 关阻尼 |
| ★ `GYRO_DAMP_SIGN` | `+1` | 阻尼符号。**开阻尼后直道反而更抖，就改 `-1`** |
| `GYRO_FILTER_ALPHA` | `0.50` | 角速度低通系数。越大响应越快，越小越平滑 |
| `GYRO_FRESH_MS` | `120` | 样本超过此时长不再参与控制 |
| `GYRO_DAMP_ERRLIM` | `6` | 仅 `|err|` ≤ 此值才施加阻尼，避免干扰过弯 |
| `GYRO_DAMP_MAX` | `150` | 阻尼项上限，防陀螺尖峰拉爆差速 |
| `GYRO_DAMP_DEADBAND` | `1.0` | 静止零偏和小噪声死区，单位 °/s |
| ★ `TURN_TARGET_DEG` | `82.0` | 转弯判定角度。转过头→调小；转不够→调大 |
| `TURN_EASE_DEG` | `30.0` | 距目标角此度数内连续减速 |
| `PIVOT_STEER` | `360` | 温和起转差速 |
| `PIVOT_STEER_MIN` | `300` | 底盘可靠转动所需的最低差速 |
| `PIVOT_STEER_BOOST` | `480` | 连续检测到低角速度后使用的坡道增力 |
| `PIVOT_MAX_TURN_MS` | `700` | 不依赖陀螺的绝对转弯时限，防止转到180° |
| `PIVOT_SLOPE_MAX_TURN_MS` | `1000` | 陀螺确认转角不足时允许的坡道时限 |
| `TURN_RATE_CRUISE` | `130.0` | 转弯前段目标角速度，单位 °/s |
| `TURN_RATE_END` | `70.0` | 接近目标角时的目标角速度，单位 °/s |
| `TURN_RATE_K` | `2.0` | 实际角速度超过目标后的降力增益 |
| `TURN_STALL_RATE` | `35.0` | 低于此角速度80ms后判定可能堵转 |
| `TURN_MIN_EXIT_DEG` | `65.0` | 有陀螺时允许计时回中退出的最小转角 |
| `TURN_MAX_DEG` | `92.0` | 即使漏检灰度也强制结束，防止转到180° |
| `TURN_REACQUIRE_MIN_MS` | `180` | 起转后至少等待此时长才接受中间四路回中 |
| `TURN_CENTER_CONFIRM_TICKS` | `1` | 角度门控后单拍回中即可确认出弯 |
| `GYRO_LOST_MS` | `300` | 超过此时间无新帧→判掉线，自动退回纯灰度 |

## 五、上车调试步骤

**第 1 步 — 验证零破坏**
先**不接**陀螺仪，直接跑。行为应与集成前完全一致（陀螺掉线 → `gyro_ok=0` → 全程走纯灰度分支）。
也可把 `GYRO_ENABLE` 设 `0` 编译，效果相同。

**第 2 步 — 接上陀螺仪**
按第一节接线，重新跑。观察：直道是否更稳、转弯是否更平滑。

**第 3 步 — 按现象调参**
- 直道**反而更抖** → 把 `GYRO_DAMP_SIGN` 改成 `-1`（符号取决于陀螺装的正反，这是最可能要改的一处）。
- 直道稳了但**反应发钝、过弯迟疑** → 调小 `GYRO_DAMP_K`。
- 转弯**转过头** → 调小 `TURN_TARGET_DEG`；**转不够或低速停顿** → 先调大 `PIVOT_STEER_MIN`，再按需要调大 `PIVOT_STEER`。

## 六、故障排查

| 现象 | 排查方向 |
|------|---------|
| 接了陀螺但无效果（一直走纯灰度） | TX/RX 是否交叉、是否共地、`UART_GYRO` 在 SysConfig 里是否无红叉 |
| 完全收不到数据 | 波特率是否 115200、陀螺是否上电（模块 LED） |
| 数据帧收到但被丢弃 | 看 `len_mismatch_count` 和 `frame_error_count`；前者表示订阅/长度不匹配，后者表示非法类型或帧格式 |
| 接收缓冲溢出 | 看 `g_gyro.overflow_count`；控制环 250Hz 调 `AS201_Poll`，正常不会溢出 |
| 编译找不到 `UART_GYRO_INST` | 打开 `empty.syscfg` 存盘一次，让 SysConfig 重新生成 `ti_msp_dl_config.h` |

> 诊断计数器（`frame_count` / `len_mismatch_count` / `frame_error_count` / `overflow_count` / `checksum_err_count`）
> 都在 `g_gyro` 句柄里，调试时用 CCS 的 Expressions 窗口直接看即可。

## 七、集成改了哪些地方（备查）

- `empty.syscfg`：配置 `UART_GYRO`（UART0 / PA28-TX / PA31-RX / 115200-8N1 / RX 中断）。
- `empty.c`：
  - 加 `#include "hlk_as201.h"`、陀螺参数区、运行时状态全局量。
  - 新增 `UART_GYRO_INST_IRQHandler`（收字节进环形缓冲）与 `gyro_update()`（解析 + 在线判定）。
  - 控制环入口调 `gyro_update()`；闭环驾驶段加直道阻尼；pivot state 2 加角度积分平滑收尾。
  - `main()` 里 `AS201_Init` + `AS201_EnsureReportConfig`（按需配置并回读验证，未接陀螺则超时后照常运行）。
- 圈数计数、pivot 状态机骨架、灰度扫描、250Hz 控制时序**均未改动**。
